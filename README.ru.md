# JT - Jira Time Tracker

CLI для логирования времени в Jira через AI. Пишите на естественном языке и подтверждайте результат перед отправкой в Jira.

## Установка

```bash
npm install -g jtw
```

## Быстрый старт

```bash
# Первичная настройка
jtw setup

# Быстрый лог через AI
jtw q "сегодня PROJ-123 разработка 2ч"

# Быстрый лог для регулярной активности
jtw q "сегодня стендап 30м"

# Интерактивный режим
jtw

# Управление шаблонами
jtw t

# Управление алиасами
jtw a
```

## Возможности

- **AI-парсинг:** Естественный язык в структурированные ворклоги с preview и подтверждением
- **Умные fallback-подсказки:** Если AI не определил задачу, `jtw` предлагает недавние задачи, историю и алиасы
- **Алиасы:** Привязка регулярной активности к задачам Jira, например `стендап -> PROJ-123`
- **Шаблоны:** Сохранение и повторное использование типовых наборов записей
- **История:** Все успешные логи сохраняются локально в SQLite и используются для подсказок
- **Периоды:** `неделю созвоны по 1.5ч каждый день` автоматически раскладывает по рабочим дням
- **Jira Server и Cloud:** Работает с on-premise и облачными инсталляциями

## Команды

| Команда | Описание |
|---------|----------|
| `jtw` | Интерактивное меню |
| `jtw setup` | Настройка Jira, AI и языка интерфейса |
| `jtw q "текст"` | Парсинг ворклога из естественного языка с подтверждением перед логированием |
| `jtw t` | Управление шаблонами |
| `jtw a` | Управление алиасами |

## Настройка

Все данные хранятся в `~/.jtw/data.db` (SQLite). Команда `jtw setup` настроит:

- URL Jira, логин, пароль
- Ключ проекта, например `PROJ`
- AI-провайдер (`Anthropic` или `OpenAI`) и API-ключ
- Язык CLI (`English` или `Russian`)

Если запустить `jtw setup` повторно, сохраненные значения будут предзаполнены. Пустые поля с секретами сохраняют текущий пароль и API-ключ.

## Типовые сценарии

### Первая настройка

```bash
jtw setup
```

После успешной проверки соединения `jtw` показывает конкретные команды, которые можно выполнить сразу.

### Быстрый AI-лог

```bash
jtw q "сегодня PROJ-123 code review 1ч, стендап 30м"
```

`jtw` парсит текст, валидирует задачи в Jira, показывает сгруппированный preview по датам и просит подтверждение перед логированием.

### Исправление unresolved task

```bash
jtw q "сегодня разбор инцидента 45м"
```

Если AI не смог определить Jira-задачу, `jtw` предложит:

- недавние задачи из локальной истории
- недавние задачи Jira, назначенные на вас
- сохраненные алиасы
- ручной ввод ключа задачи

### Workflow с алиасами и шаблонами

```bash
# Сохранять ключевые слова для регулярной активности
jtw a

# Сохранять повторяющиеся наборы записей
jtw t
```

Алиасы помогают AI точнее подбирать задачу. Шаблоны удобны для повторяющихся multi-entry сценариев.

## Примеры

```bash
# Простой лог
jtw q "сегодня стендап 30 минут"

# Несколько записей за раз
jtw q "вчера PROJ-123 разработка 6 часов, ревью 2 часа"

# AI подбирает задачу по смыслу через алиас
jtw q "сегодня созвоны 2 часа"

# Периоды раскладываются по рабочим дням
jtw q "неделю созвоны каждый день по 1.5 часа"

# Относительные даты
jtw q "позавчера фикс багов 4ч"
```

## Требования

- Node.js 18+
- Доступ к Jira Server или Jira Cloud

## Troubleshooting

**Не удается подключиться к Jira:**
Запустите `jtw setup` повторно. На этапе проверки соединения CLI теперь отдельно различает проблемы с учетными данными, сетью или VPN и SSL-сертификатами.

**AI вернул невалидный ответ:**
Повторите команду или упростите формулировку. Если проблема повторяется, попробуйте указать явный ключ задачи, например `PROJ-123`.

**better-sqlite3 не устанавливается:**
Пакет содержит нативную зависимость. Обычно prebuild скачивается автоматически. Если нет, нужен C++ компилятор:
- macOS: `xcode-select --install`
- Ubuntu: `sudo apt install build-essential python3`
- Windows: `npm install -g windows-build-tools`

## Архитектура

```text
src/
├── commands/      # CLI-команды (setup, quick, template, alias, log)
├── core/          # Ядро (store, jira, ai)
├── utils/         # Утилиты (display, menu)
├── types.ts       # TypeScript типы
└── index.ts       # Точка входа CLI
```

Минималистичный дизайн без сервисных слоев. SQLite хранит конфигурацию, алиасы, шаблоны и историю логирования.
